# ADR 0007 — Structured Output via Claude Tool Use for Brief Generation

**Status:** Accepted  
**Date:** 2026-06-23

## Context

The brief must be parsed and stored field-by-field in PostgreSQL. Asking Claude to return freeform text and then parsing it is fragile. The Claude API supports `tool_use` with a defined JSON schema, which forces structured output.

## Decision

Use `tool_choice: { type: 'tool', name: 'generate_brief' }` to force Claude to always invoke the `generate_brief` tool with a fully typed response matching the `BriefOutput` interface. The tool schema is defined in `brief-tool.definition.ts` and maps 1:1 to the `Brief` entity.

## Consequences

- Output is always valid JSON; no regex or text parsing needed.
- Adding a new brief field requires updating the tool schema, the entity, and the service — all in one place.
- The `fullBrief` field is a markdown-formatted narrative generated by Claude alongside the structured fields, usable for display without additional formatting.

## Alternatives Considered

- **Freeform text + regex parsing**: Brittle and breaks whenever Claude changes its phrasing.
- **JSON mode**: Less precise than tool use — doesn't enforce individual field types or required fields.
